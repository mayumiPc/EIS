Elevator Intelligence System (EIS) ユーザー向けガイド
Version: 1.0.1

========================================
1. このアプリについて
========================================
EIS は、画像や動画からエレベーターメーカーを推論し、設置カタログを管理するアプリです。

【AIモデルについて】
- アプリには、メーカー推論用の画像分類モデル（深層学習）が **同梱** されています。
- 推論は **お使いのPC上** で行います。推論のために画像・動画を外部のクラウドサービスへ
  自動送信する仕組みではありません（※アップデート確認など別機能での通信はあります）。

カタログ本体はアプリ内部の SQLite を使います。Access (.accdb) は取り込み時のみ使います。

========================================
2. 起動方法
========================================
- 配布フォルダ内の eis.exe を起動してください。
- 初回起動時に内部カタログがない場合は、
  「ファイル」>「カタログDBの更新…」から Access ファイルを選択して取り込んでください。

========================================
3. 推論機能（同梱AIモデルを使う）
========================================
「推論」とは、選択した画像または動画を、同梱のAIモデルに渡し、
エレベーターメーカー相当のクラス（複数候補と確率）を求める機能です。

操作の流れ:
  1) 「画像/動画を選択」でファイルを選ぶ
  2) 画面で使うモデルを選ぶ（下記「どのモデルで推論するか」）
  3) 「推論を実行」

【どのモデルで推論するか】
- **初回学習モデル（ベース）** … 製品に同梱されている既定のモデルです。すぐに利用できます。
- **ユーザー追加学習モデル** … ご自身で学習を行ったあとに使えるモデルです。
  未作成の場合はベースモデルをお使いください。

動画の場合は、動画から切り出したコマをまとめて判定します。

========================================
4. ユーザー学習機能（独自モデルの追加）
========================================
現場の画像に合わせて精度を高めたいときに、**あなた専用の追加学習モデル**を
アプリ内で作成できます。完成したモデルはファイルとして保存され、
上記の「ユーザー追加学習モデル」として推論に使えます。

操作の流れ（メイン画面の学習まわり）:
  1) 取り込み方式を選ぶ（画像ファイルを指定する／zip をまとめて指定する）
  2) 「学習用データを選択」で、学習に使う画像（または zip）を選ぶ
  3) 画像を1枚ずつ選ぶ方式では、メーカーに相当するラベル（クラス）も指定します
  4) 「ユーザー追加学習モデルを作成」で学習を開始する
  5) 完了後、アプリが案内に従って再起動すると、新しいモデルが使われます

※ zip でまとめて渡す場合は、アーカイブ内を **メーカー（クラス）ごとのフォルダ** に
  分けた構成にしてください。詳しいルールは開発者向け README を参照してください。

========================================
5. カタログ更新（Access 取り込み）
========================================
  1) 「ファイル」>「カタログDBの更新…」
  2) .accdb を選択
  3) 取り込み後、アプリは自動再起動します

========================================
6. アップデート
========================================
- 「ヘルプ」>「アップデートを確認…」で更新確認できます。
- updater.exe が見つからない場合は、次のメッセージが表示されます:
  「現在、アップデータを利用することができません。」

アップデーター利用時の注意:
- ネットワーク接続が必要です。
- 更新中はアプリが終了・再起動されます。
- updater.exe は単体実行ではなく、アプリ本体から起動してください。

========================================
7. データについて
========================================
- 内部カタログや **ユーザー学習で作られたモデル・学習データ** は、
  アプリ動作に必要なため削除しないでください。
- Access 元ファイル (.accdb) は通常利用では不要ですが、
  再取り込みする場合は再度必要です。バックアップを推奨します。

========================================
8. よくある注意点
========================================
- 学習ジョブ実行中は一部メニューが使えません。
- 起動/取り込みに失敗する場合は、関連ファイルを他アプリで開いていないか確認してください。
